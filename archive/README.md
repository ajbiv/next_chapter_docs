# Archive

This folder contains completed, outdated, or superseded documents that are no longer actively used but should be preserved for historical reference.

## Contents

- **Completed Projects**: Documentation from finished projects
- **Outdated Specifications**: Previous versions of specs that have been updated
- **Deprecated Processes**: Old workflows that have been replaced
- **Historical Decisions**: Old decision records that have been superseded

## Archiving Guidelines

### When to Archive
- Project completion
- Document superseded by newer version
- Process no longer in use
- Decision has been reversed or updated

### How to Archive
1. Move the document to the appropriate subfolder in archive
2. Add "ARCHIVED" prefix to filename
3. Update any links in active documents
4. Add archive date to document metadata

### Folder Structure
```
06-archive/
├── YYYY/                    # Archive by year
│   ├── planning/
│   ├── specifications/
│   ├── business/
│   └── meetings/
└── superseded/              # Documents replaced by newer versions
```

## Naming Convention
- `ARCHIVED-YYYY-MM-DD-original-filename.md`
- Include reason for archiving in document header

# Project Information

## Basic Details
**Project Name**: Documentation Project
**Created**: 2025-12-09
**Owner**: Alexander Bibighaus
**Purpose**: Living documentation for AI-assisted planning, specifications, and business analysis

## Project Structure
This project follows a structured approach to documentation management:

- **Planning**: Strategic documents and roadmaps
- **Specifications**: Technical requirements and designs  
- **Business**: Business analysis and process documentation
- **Meetings**: Notes, decisions, and communication logs
- **Templates**: Reusable document templates
- **Archive**: Historical and completed documents
- **Assets**: Media files and supporting materials
- **Meta**: Project configuration and standards

## Tools & Integration
- **Primary Tool**: Obsidian (markdown-based knowledge management)
- **Version Control**: Git
- **File Format**: Markdown (.md) with YAML frontmatter when needed
- **Linking**: Obsidian wikilinks `[[document-name]]`
- **Organization**: Folder-based with consistent naming conventions

## Workflow
1. **Create**: Use templates from `05-templates/`
2. **Collaborate**: Work with AI assistants to refine content
3. **Link**: Connect related documents using wikilinks
4. **Tag**: Use consistent tagging for discoverability
5. **Archive**: Move completed work to `06-archive/`
6. **Version**: Commit changes regularly to Git

## Maintenance
- Review and update templates quarterly
- Archive completed projects annually
- Clean up unused assets semi-annually
- Update documentation standards as needed

## Contact
For questions about this documentation project structure or standards, contact the project owner.
